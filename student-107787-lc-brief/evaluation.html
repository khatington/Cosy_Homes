Evaluation

Overall Conclusion:

In my opinion, I enjoyed and hated this project simultaneously throughout the whole development. As the student, I feel accomplished to see that my skills, gained from previous projects/ ALTs and hard work, have been tested to its potential in this project. I am contented with what I have and finished regardless of what I could have done more to make it better. It was a nice project to work on.

How does it meet requirements:

- The website is fully connected to the embedded system (the Micro: bit).
- You’re able to control the LEDs / heating system through the website itself.
- A visual presentation of the website is there to nearly mimic a company website.
- The values of the LED status can be seen in the database, which ensures that the right value is inputted and displayed and that there is a connection between the website and the embedded system.


How are the needs of the end user met:

- Clear information and instructions are given to the user on the website and in their account, therefore, it won’t be difficult or confusing for the user to understand.
- ALT text is added to any pictures on the website if the user is unable to see them.
- User can control the ‘heating system’ through their account by the click of a button.
- In the account, transition of colors and pictures are used to present the user that the button is either turned off or on.
- A navigation bar is set up; therefore, users can go through the website quickly and clearly.
- A login is used for the user to input their username and password.
- If username and password is wrong, Users will be alerted to ‘try again’.
- Pages such as home, contacts and products have been designed to give information to the user.


What went well:

- The overall design of the website; I enjoyed the CSS and HTML aspect of this project in general even though it was stressful.
- The setting up of the LEDs, breadboard etc.
- The connection between the database and the website went well unexpectedly.
- Due to previous projects/ ALTs that I have done in the past LC year, I was happy that I am familiar with what I had to do in the website aspect, the database connection aspect and the setting up of LEDs and breadboard aspect. Along with implementing the embedded system (the Micro: bit) to work with.


What went wrong:

- Overall, my time management wasn’t up to par as I could’ve been able to add the ‘optimal temperature’ value in my website and embedded system.
- I spent some time fixing the connection of the database to the embedded system instead due to a lot of errors.
- It was difficult to know whether equipment like the PIR sensors were working or if they’re broken.
- A lot of my time was spent on the website; the HTML pages of my website would break, and I would have to inspect the web to find the cause of the errors.


What could I have changed to if I had gone back or had more time:

- One thing I would want to do is to add that ‘optimal temperature’ input in my website and implement it into the Micro: bit.
- Have the login and sign-up page to be more sophisticated instead of assigning a username and password.
- In general, planning and time management would’ve been consistent if I were to do it again.
